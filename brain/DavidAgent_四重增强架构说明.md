# DavidAgent 四重增强架构说明

## 架构概述

DavidAgent 采用仿生双脑架构，左脑使用 **Gemini 3.1 Pro** 模型，右脑使用 **Qwen3-Max-2026-01-23** 模型。在此基础上，增加了四重增强层，形成完整的智能处理流水线。

## 四重增强层

### 1. SkillRL（本能技能层）- 最顶层
- **功能**: 处理用户输入时首先检查是否匹配已知技能模式
- **特点**: 快速响应、高准确率、低延迟
- **优先级**: 最高优先级，如果匹配则直接执行，跳过后续处理

### 2. ReasoningBank（推理避坑层）- 第二层
- **功能**: 对复杂查询进行深度推理，避免常见错误和陷阱
- **特点**: 基于历史经验的学习，包含避坑规则库
- **优先级**: 第二优先级，处理需要深度思考但不涉及记忆检索的场景

### 3. Memory Alpha（智能记忆层）- 第三层
- **功能**: 访问长期记忆和上下文信息
- **特点**: 智能记忆管理，支持语义理解和上下文关联
- **优先级**: 第三优先级，处理需要历史上下文的查询

### 4. LanceDB 7层混合检索（精确检索层）- 底层
- **功能**: 执行精确的知识检索和信息查找
- **特点**: 7层混合检索策略，确保检索结果的准确性和全面性
- **优先级**: 最低优先级，作为最后的兜底机制

## 数据流向

### 查询流程
```
用户输入 → 技能检查 → 推理检查 → 记忆检查 → 精确检索
```

### 学习流程
```
任务结果 → ReasoningBank 学习 → SkillRL 提炼 → Memory Alpha 管理
```

### 优先级原则
**技能 > 推理 > 记忆 > 检索**

## 双脑协同

- **左脑 (Gemini 3.1 Pro)**: 负责逻辑推理、事实核查、结构化知识处理
- **右脑 (Qwen3-Max-2026-01-23)**: 负责创意生成、表达优化、动态适应

两脑通过虚拟胼胝体进行高效通信和状态同步，确保系统的一致性和完整性。
