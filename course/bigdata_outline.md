# 数据驱动的智能系统架构：大纲 (Outline)

## 课程概览
*   **总学时**：56学时（理论40 + 实验16）
*   **课程定位**：本课程聚焦于数据工程架构，将智能化工具（Agentic AI）作为提升数据管道开发效率与运维质量的“副驾驶”。通过四个模块，学生将掌握从批流一体数据采集、湖仓存储治理到智能化工具应用的完整闭环。

---

## 模块一：数据采集与批流一体特征工程 (16学时)
**目标**：构建端到端的数据采集与实时特征工程链路，解决离线在线一致性挑战。

### 第1章：现代架构演进：从数据中台到智能系统
*   **子章节**：
    *   Lambda与Kappa架构优缺点分析
    *   批流一体（Streaming-Batch Integration）逻辑
    *   课程实验平台架构概览
*   **关键描述**：理解现代数据架构的演变逻辑及批流一体的核心价值。

### 第2章：数据采集基础：CDC与Kafka核心
*   **子章节**：
    *   增量采集（CDC）原理：MySQL Binlog解析
    *   Kafka核心概念：Partition, Offset, Sequence
    *   Debezium + Kafka Connect实战
*   **关键描述**：掌握基于CDC的高效增量数据采集技术。

### 第3章：实时流处理引擎：Flink SQL进阶
*   **子章节**：
    *   Flink部署模式；DataStream与SQL对比
    *   时间属性（Event Time vs Processing Time）与Watermark机制
    *   Kafka Source/Sink与Upsert-Kafka配置
*   **关键描述**：使用Flink SQL构建高效的实时处理逻辑。

### 第4章：窗口计算与状态管理
*   **子章节**：
    *   滚动、滑动、会话窗口计算
    *   状态后端（RocksDB）与Checkpoint容错
    *   解决乱序数据的Watermark策略
*   **关键描述**：深入理解流处理中的状态管理和窗口计算关键点。

### 第5章：特征工程实战：实时CTR预估特征计算
*   **子章节**：
    *   特征工程流程：采集→清洗→转换→存储
    *   实现滑动窗口计算每个商品的实时点击率
    *   写入Kafka结果Topic
*   **关键描述**：实操实时点击率特征的端到端计算流程。

### 第6章：实时特征管道：Flink + Kafka + Redis
*   **子章节**：
    *   构建端到端实时特征管道（如5分钟滑动窗口点击率）
    *   结果写入Redis供在线服务使用
*   **关键描述**：验证实时特征在NoSQL数据库中的低延迟读写。

### 第7章：批流一体特征存储：Feast基础
*   **子章节**：
    *   离线-在线一致性挑战
    *   Feast核心概念（Feature Registry, Offline/Online Store）
    *   Iceberg + Redis配置
*   **关键描述**：了解Feast在统一特征服务中的核心角色。

### 第8章：特征存储实战：Spark回填 + Flink实时写入
*   **子章节**：
    *   Spark批量回填历史特征到Iceberg
    *   Flink实时写入Redis
    *   Feast统一特征服务调用
*   **关键描述**：完成离线回填与在线更新的完整特征服务体系。

## 模块二：湖仓存储与数据治理 (12学时)
**目标**：构建统一、可治理、可追溯的湖仓一体存储，掌握数据治理与查询加速技术。

### 第9章：Iceberg核心与ACID事务
*   **子章节**：
    *   湖仓一体架构；Iceberg三层元数据原理
    *   快照隔离与MVCC；时间旅行（Time Travel）
    *   Schema演化逻辑
*   **关键描述**：掌握Iceberg的基础架构及 ACID 保证机制。

### 第10章：多引擎写入Iceberg与数据治理
*   **子章节**：
    *   Flink/Spark写入配置；乐观锁与重试机制
    *   MERGE操作实现Upsert；小文件合并（rewrite_data_files）
    *   快照过期清理（expire_snapshots）
*   **关键描述**：实战数据湖的日常维护与增量更新逻辑。

### 第11章：StarRocks湖查询加速
*   **子章节**：
    *   MPP架构（FE/BE）；向量化执行与CBO优化器
    *   Iceberg External Catalog配置
    *   谓词下推、分区裁剪与异步物化视图
*   **关键描述**：利用StarRocks实现秒级的数据湖查询加速。

### 第12章：统一数据目录与数据血缘
*   **子章节**：
    *   Hive Metastore与Iceberg REST Catalog
    *   关联结构化与非结构化数据
    *   OpenLineage实现自动化的数据血缘收集
*   **关键描述**：构建全局可见、可追溯的数据资产目录。

### 第13章：数据质量与自动化治理
*   **子章节**：
    *   Great Expectations/Deequ数据质量验证
    *   数据新鲜度监控（基于时间旅行检查分区）
    *   Ranger权限控制
*   **关键描述**：确保数据湖内容的质量、新鲜度与安全性。

### 第14章：湖仓一体综合案例
*   **子章节**：
    *   串联Iceberg, StarRocks, Feast, OpenLineage
    *   演示：时间旅行 + 物化视图 + 血缘分析
*   **关键描述**：端到端验证湖仓一体平台的综合能力。

## 模块三：数据工程的智能化工具 (12学时)
**目标**：将Agentic AI作为“副驾驶”工具，提升数据管道的开发与运维效率。

### 第15章：Harness Engineering：数据工程的智能助手
*   **子章节**：
    *   提示词工程到驾驭工程（Harness Engineering）的演进
    *   Agent Harness七大模块（工具接入、编排、记忆、安全等）
    *   开源工具：CLI-Anything, OpenCLI介绍
*   **关键描述**：理解如何将Agent作为数据工程师的自动化工具箱。

### 第16章：数据管道自动化：Agent编排与工具接入
*   **子章节**：
    *   使用LangGraph/CrewAI构建数据任务Agent
    *   将Spark SQL/Flink/Iceberg治理命令封装为Agent工具
    *   护栏机制：权限约束与人工操作审批
*   **关键描述**：实现自然语言驱动的数据任务自动执行（如“合并昨日分区小文件”）。

### 第17章：智能数据质量与自愈管道
*   **子章节**：
    *   构建数据质量Agent：自动生成Great Expectations规则
    *   检测到异常时自动触发修复流水线（重跑ETL/快照回滚）
*   **关键描述**：利用Agent实现具备“自愈”能力的数据监控逻辑。

### 第18章：Ray Data：面向AI训练的分布式数据预处理
*   **子章节**：
    *   Ray核心架构（Task/Actor）；Ray Data读取Iceberg表
    *   分布式预处理在大规模数据集上的应用
    *   Ray与Spark的互补关系
*   **关键描述**：为后续AI课程准备高效的数据预处理基础设施。

### 第19章：MLflow与DVC：数据与模型的版本管理
*   **子章节**：
    *   MLflow实验跟踪与模型注册
    *   DVC数据版本控制；关联Iceberg快照ID
    *   血缘扩展：从数据源到模型训练的端到端追溯
*   **关键描述**：掌握实现AI训练可复现性的关键技术。

### 第20章：智能化数据工程闭环与课程总结
*   **子章节**：
    *   综合案例：Agent自动检测数据倾斜→调整配置→提交PR
    *   人机协同的持续改进路径
    *   课程总结：从基础设施到AI应用的桥梁
*   **关键描述**：总结全课，展望从数据工程到AI基础设施的演进。

## 模块四：综合项目与测评 (16学时)
**目标**：构建企业级湖仓平台，并引入AI Agent辅助运维。

### 实验1：环境搭建与数据管道部署
*   **任务**：使用Helm部署Kafka, Flink, Spark, Iceberg, StarRocks, MinIO, Redis, Feast, Ray, MLflow；模拟点击流写入。

### 实验2：实时特征管道与Feast集成
*   **任务**：Flink消费Kafka计算指标写入Redis；Spark回填历史特征；Feast统一服务验证。

### 实验3：分布式数据预处理与版本管理
*   **任务**：Ray Data读取Iceberg进行预处理；MLflow记录参数；DVC管理数据集路径。

### 实验4：智能化数据治理Agent (选做/加分项)
*   **任务**：构建Agent自动监控小文件并触发合并；检测新鲜度并自动回滚告警；记录血缘。

## 课程总结与衔接
*   **基础设施支撑**：本课程构建的数据湖可直接作为后期AI课程（微调、训练）的数据源。
*   **智能赋能**：智能化工具定位于“升级版自动化脚本”，旨在降低运维门槛，提升工程质量。
